Ⅷ-11-1. 윈도우 범위 설정 <Alt-Shift-B>


윈도우 범위를 선택합니다.

센스리더에서 사용되는 윈도우 범위 영역에는 전체화면, 활성 윈도우, 클라이언트 영역, 사용자 윈도우, 포커스 윈도우가 있습니다. 여기서 윈도우 범위의 사용자 윈도우는 활성 사용자 윈도우를
의미합니다.

보다 자세한 내용은 '센스리더'에서 사용되는 개념들 > 윈도우 범위
항목을 참고하십시요.


Ⅷ-11-2. 포인터 모드 설정 <Insert-Numpad-(대시)>


포인터 모드에는 마우스 포인터와 뷰 포인터가 있습니다.

마우스 포인터는 읽기 포인터로 화면을 읽을 때 실제 마우스가 읽기 포인터와 같이 움직임으로 읽기 포인터의 위치를 시각적으로 확인할 수 있습니다. 이에 반해 뷰 포인터는 실제 마우스는 어느 한 위치에 고정되어 있고 눈에 보이지 않는 상태에서 읽기 포인터가 화면을 읽어주게 됩니다.

* 그래픽 레이블, 위치 기억 등과 같이 정안인과 같이 작업해야 하는 경우 마우스 포인터로 변경하여 사용하면 센스리더의 음성출력을 정안인은 마우스 위치를 보고 판단할 수 있어 작업이 훨씬 용이해집니다.


Ⅷ-11-3. 이동 줄 설정 <없음>


센스리더의 이동줄 기능 사용시 건너뛸 라인수를 설정합니다.

좀 더 자세한 내용은 '메뉴 구성 > 환경설정 > 이동 줄 설정'
항목을 참고하십시요.


Ⅷ-11-4. 이동 픽셀 설정 <없음>


픽셀단위 이동시 적용할 픽셀수를 설정합니다.

좀 더 자세한 내용은 '메뉴 구성 > 환경설정 > 이동 픽셀 설정' 항목을 참고하십시요.


Ⅷ-11-5. 이동 줄 읽기 방법 설정 <Ctrl-NumpadDel>


이동 줄 읽기 방법이란 수직으로 이동하는 읽기 포인터 기능에서 위치한 줄을 어떻게 읽을 것인가를 결정하는 기능입니다.

읽기 방법은 '전체 | 왼쪽 | 오른쪽 | 단어' 등 네 가지 방법이 있습니다.

전체는 이동된 줄 전체를 읽으며, 왼쪽은 현재 위치의 왼쪽 전부를, 오른쪽은 현재 위치의 오른쪽 점부를, 단어는 현재 위치의 단어만을 읽어줍니다.


Ⅷ-11-6. 캐럿 이동 줄 읽기 방법 설정 <Ctrl-Insert-NumpadDel>


캐럿이 있는 편집모드에서 위, 아래 방향키를 눌렀을 때 줄을 어떻게 읽을 것인가 선택합니다.

캐럿 이동 줄 읽기 방법은 '전체읽기 | 왼쪽읽기 | 오른쪽읽기 | 단어읽기'의 네 가지 방법이 있습니다.

* 캐럿 이동 줄 읽기 방법의 예

"여러분은 지금 센스리더의 도움말을 보고 계십니다."라는 문자열이 있고, 현재 캐럿이 '센' 글자 위치에 있다고 가정합니다.

1. 전체 읽기 : "여러분은 지금 센스리더의 도움말을 보고 계십니다."라고 라인 전체를 읽습니다.

2. 왼쪽 읽기 : "여러분은 지금"라고 음성출력을 합니다.

3. 오른쪽 읽기 : "센스리더의 도움말을 보고 계십니다."라고 읽습니다.

4. 단어 읽기 : '센스'라고 캐럿이 위치한 단어만 읽습니다.


Ⅷ-11-7. 입력 문자 읽기 방법 설정 <Ctrl-Shift-C>


키보드를 이용하여 글자를 입력할 때 음성출력 방식을 선택합니다.

입력 문자 읽기 방법에는 다음의 네 가지 읽기 방법이 있습니다.

1. 키보드로 읽기

키보드에서 입력되는 문자를 그대로 읽어줍니다.

2. 화면으로 읽기

프로그램에 따라서는 분명 한글키 입력임에도 영어를 읽어주는 경우가 있습니다. 이런 경우 '화면으로 읽기'로 입력 문자 읽기 방법을 설정해 놓으면 입력된 문자를 올바르게 읽어줍니다. 한글 2002나 이야기멀티 등에서 이 설정을 사용합니다.

3. 읽지 않기

입력 중인 문자를 읽어주지 않습니다. 이 때 어떤 음성출력 중이었다면 음성은 중지됩니다.

4. 무시

'무시'는 '읽지 않기' 방법과 동일하게 입력 문자를 읽지 않으나 다른 음성출력을 중단시키지 않는 것이 다릅니다.

통신의 채팅이나 메신저의 대화 등에서 글자 입력시 상대방의 말을 중지시키지 않음으로 MSN 메신저, 네이트온 메신저, 이야기멀티 채팅 등에서 사용하면 아주 편리한 기능입니다. 단, 컨트롤키나 기타 기능키 입력시에는 음성이 중단됩니다.


Ⅷ-11-8. 키 레이블 설정 <없음>


키 레이블 설정은 조합키나 기능키에 이름을 붙이는 기능입니다.

좀 더 자세한 내용은 '메뉴 구성 > 환경설정 > 키 레이블 설정' 항목을 참고하십시요.


Ⅷ-11-9. 기능키 설정 <없음>


센스리더의 기능키를 변경하거나 기능키가 할당되지 않은 기능에 기능키를 할당합니다.

기능키 등록은 기능별 설정과 기능키별 설정의 두 가지 기준으로 기능키 목록을 확인하며 지정할 수 있습니다.

기능키 지정은 모든 조합키에 대해 가능한 것은 아닙니다. 아래 기능키 등록 규칙을 참고하여 기능키를 지정하도록 합니다.


Ⅷ-11-10. 환경 세부 설정 <Ctrl-Shift-V>


센스리더의 세부적인 환경설정을 구성합니다.

좀 더 자세한 내용은 '메뉴 구성 > 환경설정 > 환경 세부 설정' 항목을 참고하십시요.


Ⅷ-11-11. 공백 문자 간격 설정 <없음>


정확한 단어 인식을 위해 공백문자 간격을 설정합니다.

보다 자세한 내용은 '메뉴 구성 > 환경설정 > 공백문자간격 설정' 항목을 참고하십시요.


Ⅷ-11-12. 커서 지연 시간 설정 <없음>


커서 지연시간 설정은 편집창에서 아래 방향키와 화면 스크롤이 동시에 발생할 때 생길 수 있는 문제점을 해결하기 위한 설정입니다.

보다 자세한 내용은 '메뉴 구성 > 환경설정 > 커서 지연 시간 설정' 항목을 참고하십시요.


Ⅷ-11-13. 화면 변화 읽기 방법 설정 <Ctrl-Shift-E>


화면 변화 읽기 방법은 현재 지정된 윈도우 내의 화면 변화에 대해 음성출력을 어떻게 할 것인가를 선택하는 것입니다.

아래와 같은 화면 변화 읽기 방법 중 하나를 선택합니다.

1. 읽지 않기

현재 지정된 윈도우 범위 내에서 어떤 화면의 변화가 있더라도 음성출력을 하지 않습니다.

2. 모두 읽기

현재 지정된 윈도우 범위 내에서 화면의 내용이 변경되면 그 내용을 모두 읽어줍니다.

* 윈도우즈는 화면의 구성을 유지하기 위해서 자주 화면의 내용을 갱신합니다. 이 작업은 눈으로 보아서는 알아챌 수 없을 정도로 짧은 시간에 이루어지나 화면 읽기 프로그램에서는 화면의 변화로 인식하게 됩니다. 이렇게 화면 구성을 유지하기 위한 화면 쓰기와 실제 내용 갱신에 따른 화면 쓰기를 구분하기 힘들기 때문에 '모두 읽기' 방법의 선택은 반드시 필요한 경우가 아니고서는 선택하지 않는 것이 좋습니다. 센스 리더의 기본 설정에는 이야기멀티와 새롬 데이터맨에서만 '모두 읽기'로 설정되어 있습니다.

3. 하일라이트 읽기

현재 지정된 윈도우 범위 내에서 하일라이트로 출력되는 내용만
읽어줍니다.

하일라이트는 윈도우즈가 기본으로 제공하는 '시스템 하일라이트'와 사용자가 별도로 지정해 주는 '사용자 하일라이트' 모두 해당됩니다.

4. 바뀐글자읽기

현재 지정된 윈도우 범위 내에서 새롭게 출력된 내용만
읽어줍니다.

'바뀐 글자 읽기' 방법은 실제로 내용이 변화된 부분만 읽어주기 때문에 통신의 채팅이나 메신저 프로그램에서 사용하면 새롭게 출력되는 대화 내용을 편리하게 확인할 수 있습니다.
